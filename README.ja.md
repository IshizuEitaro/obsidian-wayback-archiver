## インストール

1.  Obsidianの「コミュニティプラグイン」のブラウザから **Wayback Archiver** をインストールします。
2.  Obsidianの設定でプラグインを有効にします。
3.  プラグインの設定タブで必要なAPIキーを設定します（[グローバル API キー](#グローバル-api-キー)を参照）。

## コアコンセプト

### アーカイブ処理

プラグインは、指定された対象（現在のノート、選択されたテキスト、またはVault全体）をスキャンして、Markdownリンク（`[text](url)`、`![text](url)`）、HTMLリンク（`<a href="url">text</a>`、`<img src="url">`）、およびプレーンテキストのURL（`https://example.com`）を検出します。対象のURLごとに、Archive.org Wayback Machineの SPN (Save Page Now) API v2 を使用してスナップショットの保存を試みます。

### アーカイブリンク

アーカイブの保存に成功すると、プラグインは元のリンクの直後に新しいアーカイブリンク（MarkdownまたはHTML形式）を挿入します。このリンクの形式は設定でカスタマイズできます。

**例:**
`[Example Site](https://example.com)` が以下のように変換されます：
`[Example Site](https://example.com) [(Archived on 2025-04-10)](https://web.archive.org/web/20250410...)`

元のリンクの直後にすでにアーカイブリンクが存在する場合、「現在のノートのリンクを強制的に再アーカイブ (Force re-archive links in current note)」などの強制コマンドを使用するか、既存のアーカイブリンクが設定された「アーカイブの鮮度（Archive Freshness）」のしきい値より古い場合を除き、プラグインは不要な重複リンクの追加を自動的にスキップします。

> [!NOTE]
> **隣接リンクの判定仕様:** 既存のアーカイブリンクが元のリンクの直後 **300文字以内**（隣接検索制限）に存在する場合に「隣接」と判定されます。これにより、正規表現の処理パフォーマンスを最適化し、ドキュメントのさらに下にある無関係なアーカイブリンクとの誤判定を防ぎます。

### プロフィール

複数の設定プロフィールを作成して、異なる構成を管理できます。各プロフィールには、それぞれのルール（フィルタリング、置換、APIオプションなど）を保存できますが、Archive.org の API キーはすべてのプロフィールで共通（グローバル）のものが使用されます。設定タブから簡単にプロフィールを切り替えることができます。

### フィルタリング

いくつかのフィルタリングを使用して、どのリンクをアーカイブするかを細かく制御できます。

- **除外するパターン (Ignore Patterns):** このパターンに一致するURLは常にアーカイブをスキップします。
- **対象パターン (Include Patterns - パス、単語、URL):** アーカイブ対象となるノートやリンクを定義します。
- **パス/単語パターン (Path/Word Patterns):** **Vault全体**を対象とするコマンド、および**現在のノートを対象とするコマンド（ただしテキスト選択なしでノート全体を処理する場合）**に適用されます。ノートが処理されるためには、パスパターンと単語パターンの*両方*（両方が定義されている場合）に一致する必要があります。テキストを選択した状態でコマンドを実行した場合は、選択されたテキストが優先されるため、パス/単語パターンによるフィルタリングは無視されます。
- **URL パターン:** **すべての**コマンドに適用され、対象ノート内のリンクをフィルタリングします。

### 置換ルール

URLをアーカイブに送信する前に、検索・置換ルールを適用できます。URLの問題を修正したり、特定のバージョンのウェブサイトにリダイレクトしたりするのに便利です（例：`reddit.com` -> `old.reddit.com`）。

### アーカイブ失敗のログ

APIエラー、タイムアウト、レート制限などでリンクのアーカイブに失敗した場合、その試行がログに記録されます。これらの失敗した項目を表示、エクスポート、再試行できます。

プラグインは、内部のデータ領域に「失敗したアーカイブの内部リスト」を保持しています。このリストは、内容の確認や再試行のために、JSON または CSV 形式でエクスポートできます。エクスポートされたログファイルは独立したファイルです。プラグインデータ内の内部リストをクリアしても、以前にエクスポートしたログファイルは削除されません。エクスポートされたログファイルは、失敗ログフォルダ（通常は `.obsidian/plugins/wayback-archiver/failed_logs/`）に保存されます。

ログの肥大化を防ぐため、同じ元のURL、同じノート、同じターゲットURL、および同じ処理段階で発生した失敗は、自動的に1つの失敗エントリに合流（Coalesce）処理されます。

## コマンド

プラグインは、コマンドパレット（Ctrl/Cmd + P）からアクセスできるいくつかのコマンドを提供しています。

### 現在のノートのリンクをアーカイブ (Archive links in current note)

- **対象:** 現在アクティブなノート内のリンクを処理します。テキストが選択されている場合は、選択範囲内のリンクのみを処理します。
- **ロジック:**
    - **除外URLパターン**と**対象URLパターン**を適用します。
    - すでに隣接するアーカイブリンクが存在するリンクはスキップします（既存のリンクが「アーカイブ鮮度」設定よりも古い場合を除きます）。
    - SPN API を使用して、対象リンクのアーカイブを試みます。
    - 成功した場合、アーカイブリンクを挿入または置換します。
    - すでに最近（「アーカイブ鮮度」設定内）のスナップショットが存在するかレート制限に達していることをAPIが示した場合は、隣接するアーカイブリンクが存在しない場合に限り、**新規保存ではなく、すでに保存されている履歴の中から既存の最新スナップショットURL（タイムスタンプ付き）を検索・取得**して挿入を試みます。
- **注意:** パス/単語パターンによるフィルタリングは、**テキスト選択をせずノート全体を処理する場合にのみ適用されます**。テキストを選択してコマンドを実行した場合は、選択範囲の内容が優先されるため、パス/単語パターンは無視されます。

### Vault内のすべてのリンクをアーカイブ (Archive all links in vault)

- **対象:** Vault全体のすべてのMarkdownファイル内のリンクを処理します。実行前に確認を求められます。
- **ロジック:**
    - **対象パスパターン**と**対象単語パターン**に基づいてノートをフィルタリングします（両方が定義されている場合は両方に一致する必要があります）。スキップされたファイルはログに記録されます。
    - 選択されたノート内のリンクを、**除外URLパターン**と**対象URLパターン**に基づいてフィルタリングします。
    - 「現在のノートのリンクをアーカイブ」と同様のアーカイブおよび挿入/置換ロジックを適用します。
- **注意:** Vaultのサイズやリンクの数によっては、処理にかなりの時間がかかる場合があります。

### 現在のノートのリンクを強制的に再アーカイブ (Force re-archive links in current note)

- **対象:** 現在アクティブなノート内のリンクを処理します。テキストが選択されている場合は、選択範囲内のリンクのみを処理します。
- **ロジック:**
    - **除外URLパターン**と**対象URLパターン**を適用します。
    - 「アーカイブ鮮度」設定や既存のアーカイブリンクの有無を**無視して**、対象リンクのアーカイブを試みます。
    - 成功した場合、既存の隣接するアーカイブリンクを**置換**するか、存在しない場合は新しく挿入します。
    - アーカイブが失敗するかレート制限に達した場合、**リンクの挿入や置換は行われません**。既存のリンクがある場合はそのまま維持されます。
- **注意:** パス/単語パターンによるフィルタリングは、**テキスト選択をせずノート全体を処理する場合にのみ適用されます**。テキストを選択してコマンドを実行した場合は、選択範囲の内容が優先されるため、パス/単語パターンは無視されます。

### Vault内のすべてのリンクを強制的に再アーカイブ (Force re-archive all links in vault)

- **対象:** Vault全体のすべてのMarkdownファイル内のリンクを処理します。実行前に確認を求められます。
- **ロジック:**
    - **対象パスパターン**と**対象単語パターン**に基づいてノートをフィルタリングします。スキップされたファイルはログに記録されます。
    - 選択されたノート内のリンクを、**除外URLパターン**と**対象URLパターン**に基づいてフィルタリングします。
    - 「現在のノートのリンクを強制的に再アーカイブ」と同様のアーカイブおよび置換ロジックを適用します。
- **注意:** 処理にかなりの時間がかかる場合があります。

### 失敗したアーカイブを再試行 (Retry failed archive attempts)

- **アクション:** 失敗ログフォルダ（通常は `.obsidian/plugins/wayback-archiver/failed_logs/`）から、失敗ログファイル（`.json` または `.csv`）を選択するよう求められます。
- **ロジック:** 選択されたファイルにリストされている各URLのアーカイブを試みます。
- **成功時:**
    - ログファイルから該当のエントリを削除し、**隣接するアーカイブリンクがまだ存在しない場合に限り**、元のノートにアーカイブリンクを挿入します。
    - すべてのエントリが成功した場合、ログファイルは自動的に削除されます。
    - 「失敗ログの自動クリア」が有効になっていない限り、ログをクリアする前に確認を求められます。

### 失敗したアーカイブを再試行（強制置換） (Retry failed archive attempts (Force replace))

- **アクション:** 標準の再試行と同じですが、強制ロジックを使用します。
- **ロジック:** 選択されたファイルにリストされている各URLのアーカイブを試みます。
- **成功時:**
    - ログファイルから該当のエントリを削除し、元のノートにある既存の隣接するアーカイブリンクを**強制的に置換**（存在しない場合は新規挿入）します。
    - すべてのエントリが成功した場合、ログファイルは自動的に削除されます。
    - 「失敗ログの自動クリア」が有効になっていない限り、ログをクリアする前に確認を求められます。

### 失敗したアーカイブのログをエクスポート (Export failed archive log)

- **アクション:** 現在の失敗したアーカイブのリストを新しいファイルにエクスポートします。
- **ロジック:**
    - CSV または JSON 形式のいずれかを選択するよう求められます。
    - 失敗ログフォルダ（通常は `.obsidian/plugins/wayback-archiver/failed_logs/`）内に、タイムスタンプ付きのファイル（例：`wayback-archiver-failed-log-YYYYMMDDHHMMSS.json`）としてログを保存します。
    - エクスポートが成功した後、内蔵の失敗ログリストをクリアするかどうか確認を求められます。

### 失敗したアーカイブのログをクリア (Clear failed archive log)

- **アクション:** 確認後、内蔵の失敗したアーカイブのリストをクリアします。
- **注意:** エクスポート済みのログファイルは削除されません。

### archive.today および ウェブ魚拓の補助コマンド

これらのコマンドを使用すると、代替プロバイダーと直接連携したり、バックグラウンドの保留キューを手動で処理・管理したりできます。

- **Submit current note links to archive.today (現在のノートのリンクを archive.today へ送信)**:
    - **対象:** 現在アクティブなノートのリンク。テキストが選択されている場合は、選択範囲内のリンクのみ。
    - **処理:** ノート/選択範囲内の対象リンクについて、archive.today へのバックグラウンド送信を開始します。最終的なスナップショットURLがすぐに得られない場合は、保留キューに追加されます。
    - **注意:** このコマンドは archive.today の非公式な送信フローに依存しており、レート制限、CAPTCHA、ボット対策、サイト仕様変更などにより失敗する可能性があります。
- **Insert latest archive.today snapshot in current note (現在のノートに最新の archive.today スナップショットを挿入)**:
    - **対象:** 現在のノートのリンク。テキストが選択されている場合は、選択範囲内のリンクのみ。
    - **処理:** 対象リンクの最新の既存スナップショットを archive.today に問い合わせ、新規保存を行わずに、その場でアーカイブリンクを挿入します。
- **Insert latest Web Gyotaku snapshot in current note (現在のノートに最新のウェブ魚拓スナップショットを挿入)**:
    - **対象:** 現在のノートのリンク。テキストが選択されている場合は、選択範囲内のリンクのみ。
    - **処理:** 対象リンクの最新の既存スナップショットをウェブ魚拓 (Megalodon.jp) に問い合わせ、新規保存を行わずに、その場でアーカイブリンクを挿入します。
- **Check pending archive.today snapshots now (保留中の archive.today スナップショットを今すぐ確認)**:
    - **処理:** 保留中になっているすべての archive.today バックグラウンド送信リクエストのステータス確認ポーリングサイクルを、今すぐ強制的に実行します。
- **Open next failed URLs in archive.today (失敗した次のURLを archive.today で開く)**:
    - **処理:** 保存に失敗したアーカイブ対象のURLの次のバッチを、デフォルトのブラウザで直接 archive.today の手動保存ページで開きます。これにより、CAPTCHAの解決や手動での保存が簡単になります。
- **Open next failed URLs in Web Gyotaku (失敗した次のURLをウェブ魚拓で開く)**:
    - **処理:** 保存に失敗したアーカイブ対象のURLの次のバッチを、デフォルトのブラウザで直接ウェブ魚拓 (Megalodon.jp) の手動保存ページで開きます。
    - **注意:** プラグインはウェブ魚拓へURLを自動送信しません。保存操作が必要な場合は、ブラウザ上でユーザーが手動で完了する必要があります。

## 設定ガイド

設定は、Obsidianの設定 -> コミュニティプラグイン -> Wayback Archiver からアクセスできます。

### グローバル API キー

これらは Archive.org SPN API v2 を使用するために必要なキーであり、すべてのプロフィールで共有されます。

- **Archive.org SPN Access Key:** S3形式のアクセスキーを入力します。
- **Archive.org SPN Secret Key:** S3形式のシークレットキーを入力します。
- **キーの取得方法:** Archive.org にログインした状態で、アカウント設定ページ（[https://archive.org/account/s3.php](https://archive.org/account/s3.php)）から API キーを生成・取得できます。

### プロフィールの管理

異なる設定のセット（プロフィール）を管理します。

- **Active Profile:** 現在アクティブで、編集中のプロフィールをドロップダウンから選択します。
- **Create Profile:** デフォルト設定をベースに新しいプロフィールを作成し、それを有効にします。名前の入力を求められます。
- **Rename Profile:** 現在アクティブなプロフィールの名前を変更します（"default" プロフィールは変更できません）。
- **Delete Profile:** 確認後、現在アクティブなプロフィールを削除します（"default" プロフィールは削除できません）。削除後は "default" プロフィールに切り替わります。

### プロフィール設定

これらの設定は、**現在アクティブなプロフィールにのみ**適用されます。

#### 一般設定 (General)

- **Date Format:** アーカイブリンク内の `{date}` プレースホルダーに使用する日付形式を定義します。`date-fns` の日付フォーマットトークン（例: `yyyy-MM-dd`, `dd MMM yyyy`, `yyyy/MM/dd`）を使用します。デフォルト: `yyyy-MM-dd`。
- **Archive Link Text:** 挿入されるアーカイブリンクのテンプレート。以下のプレースホルダーが利用可能です。
    - `{date}`: フォーマットされたアーカイブ日付に置換されます。
    - `{provider}`: アーカイブプロバイダー名（例: `Wayback Machine`, `archive.today`, `ウェブ魚拓 (Megalodon.jp)`）に置換されます。
    - **デフォルト:** `(Archived on {date})`。

#### フィルタリングルール (Filtering Rules)

どのノートやリンクを処理するかを制御します。

- **Ignore URL Patterns:** アーカイブ処理中にスキップするURLパターンを1行に1つずつ入力します（簡易テキストまたは正規表現に対応）。デフォルトで `web.archive.org` は無視されます。
- **URL Patterns:** (すべてのコマンドに適用) このパターンのいずれかに一致するURLを持つリンクのみを処理します（1行に1つ、簡易テキストまたは正規表現）。空欄にすると、すべてのURLのリンクを処理します（無視パターンが優先されます）。
- **Path Patterns:** (Vault全体のコマンド、およびテキスト選択なしで現在のノートを処理するコマンドに適用) ファイルパスがこれらのパターンのいずれかに一致するノートのみを処理します（1行に1つ、簡易テキストまたは正規表現）。空欄にすると、すべてのパスのノートを処理します。
- **Word/Phrase Patterns:** (Vault全体のコマンド、およびテキスト選択なしで現在のノートを処理するコマンドに適用) これらの単語またはフレーズの少なくとも1つを含むノートのみを処理します（1行に1つ、簡易テキスト検索）。空欄にすると、内容に関わらずすべてのノートを処理します。
- **適用ロジック:** 対象となるコマンド（Vault全体、および選択なしでの現在のノート処理）において、パスパターンと単語パターンの両方が定義されている場合、ノートが処理対象に含まれるためには**両方**に一致する必要があります。その上で、URLパターンによってノート内の各リンクがフィルタリングされます。

#### URL 置換ルール (URL Substitution Rules)

URLをアーカイブに送信する*前*に、検索・置換ルールを適用します。

- 「Add Substitution Rule」をクリックすると新しい行が追加されます。
- **Find:** 検索するテキストまたは正規表現を入力します。
- **Replace with:** 置換後のテキストを入力します。一致した部分を削除したい場合は空欄のままにします。
- **Regex?:** 「Find」に入力したパターンを正規表現として扱う場合にチェックを入れます（グローバル検索フラグ `g` が自動的に適用されます）。
- **Remove:** その置換ルールを削除します。
- ルールは表示されている順（上から順）に適用されます。

#### アーカイブプロバイダーとフォールバック (Archive Providers and Fallbacks)

メインの Wayback Machine によるアーカイブが失敗した際の、代替ソースを設定します。

- **Use archive.today fallback:** Wayback Machine での保存失敗時に、既存の archive.today スナップショットを検索・解決して挿入します。この設定単体では、新規の archive.today 保存リクエストは送信されません。
- **Use Web Gyotaku fallback:** 先行プロバイダーがすべて失敗した際、既存のウェブ魚拓 (Megalodon.jp) スナップショットを解決して挿入します。この機能は解決のみ（Resolve-only）であり、自動保存は実行されません。
- **Background archive.today auto-submit:** 【実験的機能】デフォルトでは無効です。有効にすると、既存のスナップショットを解決するだけでなく、Wayback Machine の保存失敗時に、既存のスナップショットを探す前にバックグラウンド HTTP リクエスト経由で archive.today へ保存リクエストを自動送信します。

#### archive.today 保留キューおよび手動フォールバックオプション (archive.today Pending Queue and Manual Fallback Options)

archive.today への実験的なバックグラウンド送信・検証キューの細部を調整します。

- **archive.today submit delay between requests (ms):** archive.today での IP レート制限を回避するため、自動送信リクエストの間隔をミリ秒単位で設定します。設定範囲: 1,000ms 〜 10,000ms。デフォルト: 5,000ms。
- **archive.today pending poll interval (ms):** バックグラウンドスケジューラが、保留中になっている archive.today 送信のステータスをチェックする間隔を設定します。設定範囲: 15,000ms 〜 300,000ms。デフォルト: 60,000ms。
- **archive.today pending poll batch size:** 1回のステータス確認ポーリングサイクルで確認する、保留中エントリーの数を設定します。設定範囲: 1 〜 10。デフォルト: 3。
- **archive.today pending max wait (ms):** 保留中の送信がタイムアウトになり、失敗リストに移動されるまでの最大待ち時間を設定します。設定範囲: 60,000ms 〜 1,200,000ms。デフォルト: 600,000ms。
- **archive.today max pending count:** バックグラウンドの自動送信キューにローカルで同時保持できる保留アイテムの最大数です。これはリクエスト頻度を直接制御するものではありません（リクエスト頻度は送信遅延、ポーリング間隔、ポーリングバッチサイズによって制御されます）。キューが上限に達した場合、既存の保留エントリーが解決・失敗・クリアされるまで、新規の自動送信リクエストはキューイングされずスキップされるか直接失敗として扱われます。設定範囲: 1 〜 100。デフォルト: 30。
- **Manual save batch size:** 手動でフォールバックを開くコマンドを実行した際、一度にブラウザのタブで開くURLの最大数を設定します。設定範囲: 1 〜 5。デフォルト: 5。

#### URLごとのアーカイブポリシー (Per-URL Archive Policies)

ターゲットのURLに対する正規表現パターンに基づいて、使用するアーカイブプロバイダーを細かくカスタマイズします。

- **フォーマット:** `パターン => プロバイダー` の形式で、1行に1ルールを入力します。
- **サポートされているプロバイダー:**
    - `wayback`: Archive.org Wayback Machine（保存および解決）
    - `archiveToday`: archive.today（既存スナップショットの解決のみ、新規保存は行いません）
    - `archiveToday:auto`: archive.today（実験的なバックグラウンド自動送信を含み、後で利用可能な場合に解決を試みます。有効にする前に [【実験的機能】archive.today 自動送信 (auto-submit)](#実験的機能archivetoday-自動送信-auto-submit) をご確認ください。）
    - `megalodon`: ウェブ魚拓 (Megalodon.jp)（既存スナップショットの解決のみ、自動送信は非対応）
- **設定例:**
    ```text
    ^https://x\.com/ => archiveToday:auto
    example\.com => wayback, archiveToday
    ```

#### 詳細設定 (Advanced Settings)

パフォーマンスや挙動を微調整します。

- **API Request Delay (ms):** 連続するAPI呼び出し（キャプチャ開始、ステータス確認、次のリンクの処理など）の間で待機する最小時間（ミリ秒単位）。レート制限（429エラー）を回避するのに役立ちます。デフォルト: `2000ms`（2秒）。
- **Max Status Check Retries:** 保留中のアーカイブジョブのステータスをチェックする最大回数。この回数に達すると諦めて失敗（Timeout）として記録します。デフォルト: `3`。
- **Archive Freshness (Days):** （標準のアーカイブコマンドのみ）リンク直後に隣接するアーカイブリンクが存在し、そのタイムスタンプが指定日数以内であれば、そのリンクはスキップされます。隣接するアーカイブリンクがこのしきい値より古い場合、または日付を判定できない場合、プラグインは新しいアーカイブをリクエストし、既存の隣接アーカイブリンクを置換することがあります。`0` に設定すると、既存の隣接アーカイブリンクは原則として古いものとして扱われます。デフォルト: `0`。
- **Auto Clear Failed Logs:** 有効にすると、再試行して成功した失敗ログエントリが、確認なしで内部の失敗ログから自動的に削除されます。デフォルト: `false`。

#### SPN API v2 オプション

Archive.org SPN API v2 のキャプチャプロセスに関する特定の設定を制御します。詳細は [API ドキュメント](https://docs.google.com/document/d/1Nsv52MvSjbLb2PCpHlat0gkzw0EvtSgpKHu4mk0MnrA/edit) をご覧ください。

- **Capture Screenshot:** スナップショットキャプチャ時にスクリーンのスクリーンショット画像も要求します（`capture_screenshot=1`）。デフォルト: `false`。
- **Capture All Resources (capture_all=1):** CSS、JS などの埋め込みリソースをより多く取得し、エラーハンドリングを向上させます（`capture_all=1`）。キャプチャ時間が増加する可能性があります。デフォルト: `false`。
- **JS Behavior Timeout (ms):** キャプチャ中の JavaScript 実行を許可する最大時間（ミリ秒）。`0` を指定すると、API のデフォルト値が使用されます（`js_behavior_timeout`）。デフォルト: `0`。
- **Force GET Request (force_get=1):** アーカイバーに強制的に HTTP GET リクエストを使用させます（`force_get=1`）。デフォルト: `false`。
- **Capture Outlinks (capture_outlinks=1):** 対象のURL*からリンクされている*外部ページも合わせてキャプチャを試みます（`capture_outlinks=1`）。非常に時間がかかり、多くのリソースを消費する可能性があるため注意して使用してください。デフォルト: `false`。

## トラブルシューティング FAQ

**Q: APIキーが機能しない、または「Configuration Error」と表示されます。**

**A:**

1.  [https://archive.org/account/s3.php](https://archive.org/account/s3.php) からコピーした **Access Key** と **Secret Key** が、プラグイン設定の **Global API Keys** セクションに正しく貼り付けられているか確認してください。
2.  前後に余分なスペース（空白文字）が入っていないことを確認してください。
3.  これらのキーはグローバル設定です。プロフィールごとに個別に設定する必要はありません。

**Q: アーカイブが頻繁に失敗する、またはレート制限エラー（HTTP 429など）が発生します。**

**A:**

1.  Archive.org APIには利用制限があります。詳細設定の **API Request Delay** を引き上げてみてください（例: `3000ms` または `5000ms`）。
2.  Archive.org が設けている1日のキャプチャ上限に達している可能性があります。プラグインは、標準アーカイブ処理中に 429 エラーや「too many captures」エラーを検出した場合、自動的に利用可能な最新のスナップショットURLの取得を試みることで、この問題に対応しています。
3.  エクスポートコマンドや再試行コマンドを使用して、**Failed Archive Log** を確認し、エラーメッセージの詳細を調べてください。

**Q: 特定のリンクがアーカイブされません。なぜですか？**

**A:** 以下の点を確認してください：

1.  **除外パターン (Ignore Patterns):** アクティブなプロフィールの「Ignore URL Patterns」設定で、そのURLがパターンに一致していませんか？
2.  **対象パターン (Include Patterns):**
    - （Vault全体、またはテキスト選択なしで現在のノートを処理する場合）ノートのパスが「Path Patterns」に一致していますか？
    - （Vault全体、またはテキスト選択なしで現在のノートを処理する場合）ノートの内容が「Word/Phrase Patterns」に一致していますか？
    - （すべてのコマンド）リンクのURLが「URL Patterns」に一致していますか？設定ガイドに記載されたフィルタリングの優先ロジックを確認してください。
3.  **隣接するアーカイブリンク:** 元のリンクの直後にすでにアーカイブリンクが存在していませんか？標準コマンドは、既存のリンクが「Archive Freshness」設定よりも新しければスキップします。強制再アーカイブコマンド（「現在のノートのリンクを強制的に再アーカイブ (Force re-archive links in current note)」など）であれば処理を実行できます。
4.  **非HTTP(S)スキーム:** プラグインは `http://` および `https://` リンクのみを対象とします。`ftp://` や `mailto:` などの他のプロトコルは無視されます。
5.  **APIエラー:** サイト自体の理由で Archive.org 側で保存に失敗している可能性があります。「Failed Archive Log」を確認してください。

**Q: 強制再アーカイブコマンド（「現在のノートのリンクを強制的に再アーカイブ (Force re-archive links in current note)」など）を使用しましたが、古いアーカイブリンクが置換されませんでした。**

**A:** これらの強制再アーカイブコマンドは、**新しくリクエストしたアーカイブの保存に成功した場合にのみ**既存のリンクを置換します。保存に失敗した、またはレート制限に達した場合は、安全のためノートの内容は書き換えられず、古いリンクがそのまま残されます。

**Q: アーカイブに毎回失敗するリンクはどう管理すればよいですか？**

**A:**

1.  **失敗したアーカイブのログをエクスポート** コマンドを使用して、失敗したURLとエラーの一覧（CSV または JSON）を出力します。
2.  なぜ失敗するのか原因を調査します（Webサイトがアーカイブ用のクローラーをブロックしている、一時的なサーバーダウンなど）。
3.  定期的に **失敗したアーカイブを再試行** コマンドを実行して、再トライします。
4.  どうしても保存できないURLの場合は、そのURLを「Ignore URL Patterns」に登録することを検討してください。
5.  追跡を止めたい場合は、**失敗したアーカイブのログをクリア** コマンドを使用してリストから削除できます。

**Q: 置換ルール（Substitution Rules）がうまく機能しません。**

**A:**

1.  正規表現を使用している場合は、**Regex?** にチェックが入っていること、および正規表現の構文が正しいか確認してください（グローバル検索フラグ `g` は自動的に適用されます）。
2.  ルールの順序を確認してください。ルールは上から順番にシーケンシャルに適用されます。
3.  「Find」のパターンが、置換対象となるURLの部分に正確に一致しているか確認してください。通常のテキスト一致は、大文字と小文字を区別します。

**Q: 現在のノートをアーカイブする際、パスパターンや単語パターンによるフィルタリングはどう適用されますか？**

**A:** パスパターンと単語パターンは、**現在のノートのコマンドであっても、テキスト選択をせずに「ノート全体」を処理する場合**に適用されます。もし現在のノートがこれらのパターンに一致しない（パスが一致しない、または指定された単語が含まれていない）場合、アーカイブ処理はスキップされます。
一方で、**テキストを選択した状態でコマンドを実行した場合**は、選択範囲の内容を処理することが優先されるため、パスパターンや単語パターンによるノート自体のフィルタリングは無視され、選択範囲内のリンクがアーカイブされます。なお、URLパターン（URL Patterns）によるリンク単体のフィルタリングは、選択の有無に関わらずすべての処理で機能します。

**Q: archive.today 自動送信でスナップショットが作成されません。なぜですか？**

**A:** archive.today 自動送信は実験的かつベストエフォートな機能です。archive.today は安定した公式保存 API を提供していないため、レート制限、CAPTCHA、ボット対策、地域・ネットワーク差、サイトの仕様変更などによってブロックされる可能性があります。必要に応じて **Open next failed URLs in archive.today** コマンドでブラウザから手動保存し、その後 **Check pending archive.today snapshots now** または **Insert latest archive.today snapshot in current note** を実行してください。

## 制限事項

- **Wayback Machine:** キャプチャの保存が失敗、タイムアウト、またはレート制限される可能性があります。保存制限に達した場合、プラグインは新規保存の代わりに、利用可能な最新の既存スナップショットの挿入を試みる場合があります。
- **archive.today:** 既存スナップショットの解決が可能であり、実験的な自動送信も利用できます。ただし、archive.today は安定した公開保存 API を提供していないため、自動送信機能はベストエフォート（最善努力）であり、予告なく機能しなくなる可能性があります。
- **ウェブ魚拓 (Megalodon.jp):** 既存スナップショットの解決（検索）のみをサポートしています。本プラグインからウェブ魚拓へ新規URLを自動的に保存・送信する機能はありません。
- **隣接アーカイブの判定仕様:** 既存のアーカイブリンクは、元のリンクの直後の隣接検索制限文字数の範囲内のみで検出されます。
- **URLの重複:** 同じURLがノート内に複数存在する場合、プラグインはスキャン時に検出された元の位置に最も近いものを処理します。これは通常の編集に対しては十分に堅牢ですが、アーカイブ処理中にノートが大幅に編集された場合はベストエフォート動作となります。

### 【実験的機能】archive.today 自動送信 (auto-submit)

`archive.today 自動送信` は実験的なフォールバック機能です。有効にすると、Wayback Machineでの保存失敗時に、設定に応じて自動的にURLを archive.today へ送信して保存を試みます。

archive.today は自動保存用の安定した公開APIを提供していないため、この機能はベストエフォート（最善努力）でのみ提供されます。具体的には、以下のような要因により、送信が失敗したり動作が不安定になったりする可能性があります：

- archive.today 側のウェブサイト仕様の頻繁な変更
- 厳しいIPレート制限や、ボット排除（CAPTCHAなど）のチェック
- 地域やネットワーク環境による接続性の違い
- スナップショット生成の遅延（タスクが保留中キューに入り、完了まで時間がかかる場合があります）

このオプションはデフォルトで無効になっています。より確実で安定したアーカイブ保存を行いたい場合は、手動の archive.today 保存コマンドを使用することをお勧めします。

> [!WARNING]
> この機能はベストエフォートであり、予告なくいつでも機能しなくなる可能性があります。有効化されている場合でも、スナップショットが必ず作成されることや、最終的なURLが取得できることは保証されません。これらの制約を十分に理解した上でのみ有効化してください。
